description: Capture a post-task retrospective about SuPi tooling in this repo

You have just completed a task in this repository. Produce a compact markdown retrospective focused on how SuPi tooling helped, fell short, or added unnecessary context during that task.

## Scope

- Focus on the just-finished task, not the entire SuPi suite in the abstract.
- Ground every claim in direct session experience when possible.
- If you infer that a tool would have helped but did not use it, label that clearly as an inference.
- If no SuPi-specific tools were used, say so explicitly and focus on which available tools would have helped.

## What to evaluate

- Which SuPi tools were used and how they helped or fell short.
- Which tools or guidance would likely have helped but were not used.
- Whether missed help was caused by discoverability, weak guidance, or missing examples.
- Missing utilities or missing features in existing tools.
- Unhelpful, redundant, badly timed, or token-wasting context, instructions, or tool output.

## Output rules

- Keep the result concise and specific.
- Separate direct experience from inference.
- Prioritize the highest-value improvements first.
- Do not edit files, open issues, update OpenSpec artifacts, or take any follow-up action.
- Stop after producing the retrospective.

## Required output

## SuPi Tooling Retrospective

**Task completed**: <1-2 sentence summary>

### Tools used
- `<tool or prompt>` — what helped, what did not, and whether this is direct experience

### Missed opportunities
- Tools or guidance that would likely have helped but were not used, including whether this was a discoverability problem

### Missing pieces
- Missing utility, missing feature, or missing documentation/example

### Unhelpful or noisy context
- Guidance, prompt text, or tool output that added little value or inflated context unnecessarily

### Prioritized recommendations
1. <highest-value improvement>
2. <next improvement>
3. <next improvement>

### Confidence / evidence
- What is based on direct usage vs. inference
