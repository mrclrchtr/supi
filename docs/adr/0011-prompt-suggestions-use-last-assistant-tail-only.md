# Prompt suggestions use only the last assistant tail

`@mrclrchtr/supi-prompt-suggestions` will send only a package-owned system prompt plus the last 8,000 characters of the final assistant text to the suggestion model. The system prompt is a short instruction to produce exactly one plausible next user prompt from the provided assistant message. It will not name PI, SuPi, the conversation, or the current project. The model receives no transcript, tool outputs, file contents, current editor text, cwd, session name, tool list, or PI system-prompt metadata. The suggestion model is asked for exactly one plain-text user prompt, not JSON and not multiple alternatives. The returned suggestion is normalized to a single line by trimming whitespace, stripping simple wrapping quotes, and collapsing internal newlines to spaces; empty outputs are rejected; outputs over 240 characters are truncated instead of rendered. This keeps background advisory generation cheap, privacy-bounded, and aligned with the product intent: suggest a useful next user prompt from the last assistant response.
