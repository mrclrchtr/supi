# Prompt suggestion generation runs in the background

`@mrclrchtr/supi-prompt-suggestions` will start suggestion generation from `agent_end` without awaiting the model call, using its own `AbortController` and generation id to cancel or discard stale results. If a new `agent_end` fires while a previous generation is still in-flight, the extension aborts the previous call and starts fresh generation for the latest assistant response. The model call has a 10 s timeout; if the model does not respond in that window, generation is cancelled and the status spinner is replaced with a concise error message. On other failures (network error, auth failure), the spinner is likewise replaced with an error indication rather than silently dropped. Any manual editor change dismisses the current or pending prompt suggestion for that assistant response, so stale ghost text does not reappear after the user starts writing. Generation is skipped entirely when the last assistant message contains no text content blocks (e.g., pure tool-call responses). Awaiting inside `agent_end` would keep PI's agent run non-idle until the advisory suggestion finishes; background generation preserves PI responsiveness, while an extension-owned status spinner makes the remaining work visible without reusing PI's main working indicator.
