# Task 1: Refactor supi-tree-sitter so split tree_sitter_* tools derive from one shared metadata source

Create `packages/supi-tree-sitter/src/tool/tool-specs.ts` as the single source of truth for the split `tree_sitter_*` tool surface: tool names, descriptions, prompt snippets, prompt guidelines, and parameter schemas. Update `packages/supi-tree-sitter/src/tool/register-tools.ts` and `packages/supi-tree-sitter/src/tool/guidance.ts` to derive from that metadata instead of hardcoding literals. Keep `packages/supi-tree-sitter/src/tree-sitter.ts` thin after the refactor. Retire `packages/supi-tree-sitter/src/tool/action-specs.ts` as the public metadata source; remove it or reduce it to compatibility helpers only if a smaller internal helper still earns its existence. Update `packages/supi-tree-sitter/README.md` so it documents the actual split `tree_sitter_outline`, `tree_sitter_imports`, `tree_sitter_exports`, `tree_sitter_node_at`, `tree_sitter_query`, and `tree_sitter_callees` tools instead of the old multiplexed `tree_sitter` tool. Cover the refactor with `packages/supi-tree-sitter/__tests__/guidance.test.ts`, `packages/supi-tree-sitter/__tests__/tool-focus.test.ts`, and `packages/supi-tree-sitter/__tests__/tool.test.ts`.
