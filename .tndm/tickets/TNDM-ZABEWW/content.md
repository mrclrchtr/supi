Preserve the existing cp -RL staging plus npm pack pipeline because it dereferences pnpm workspace symlinks and avoids bad bundled tarball paths seen with direct pnpm pack for the meta-package. Add a pre-pack staged-manifest export step that makes all shipped package.json files npm-compatible: resolve pnpm workspace: specs to concrete workspace versions using pnpm-compatible rules, strip devDependencies from staged publish manifests to avoid private workspace test utilities leaking, keep bundledDependencies arrays intact, and verify packed tarballs contain no workspace: protocol in any package.json. Expand verification so tarballs are checked for ../ paths, clean manifests, extraction success, and ideally installability.
