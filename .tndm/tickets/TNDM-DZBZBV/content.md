renderNoteStatus() in ui-rich-render-notes.ts renders the full note text on a single line without wrapping or truncating. When a user types a long note (e.g., ~184 chars), the rendered line " Notes: <note>" can exceed the terminal width (e.g., 192 > 188), crashing pi with: "Rendered line exceeds terminal width". Fix by passing width and wrapping the note text with wrapTextWithAnsi.
