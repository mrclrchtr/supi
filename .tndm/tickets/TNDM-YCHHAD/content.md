When installing @mrclrchtr/supi via `pi install npm:@mrclrchtr/supi`, npm 7+ auto-installs peer dependencies. Since pi-tui, pi-ai, typebox, and pi-agent-core are listed as peer deps of bundled sub-packages but are already provided by the pi runtime (and not present in npm's global node_modules root when installed via mise), npm tries to install them fresh. This pulls in pi-tui's dependency koffi, a native addon whose install script fails on Node.js 24. The fix is to mark these pi-provided peer deps as optional via peerDependenciesMeta, so npm doesn't try to auto-install them.
